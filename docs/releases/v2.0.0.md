# zsh-config v2.0.0 — macOS-only rebuild

Complete rebuild for macOS (Apple Silicon & Intel). **BREAKING:** all WSL and
Ubuntu/Debian support removed — Linux users should stay on v1.1.1.

## Install / upgrade

```bash
git clone https://github.com/nandordudas/zsh-config ~/.config/zsh
~/.config/zsh/install.sh --git-name "Your Name" --git-email "you@example.com"
```

Upgrading from v1.x: re-run `install.sh` after pulling; it is idempotent and
backs up existing files. Uninstall anytime with `uninstall.sh` (restores backups).

## Highlights

- **One-command install AND uninstall** with a tested round-trip: originals
  (`~/.zshenv`, `tmux.conf`) are backed up on install and restored on uninstall
- **Brewfile + Brewfile.dev** (`brew bundle`) declare all packages; jq added
- **mise** manages runtimes: node pinned globally (npm tools in
  `~/.default-npm-packages` survive node upgrades); go/python per-project via
  `.mise.toml`; rust per-project via `rust-toolchain.toml` (rustup manager
  only, no global toolchain); direnv removed per mise upstream guidance
- **Git overhaul**: per-host identities now actually load (`includeIf` was
  missing — silently broken in v1.x), branch-agnostic aliases via
  `git default-branch`, macOS FSEvents fsmonitor, all-core fetch/index
  parallelism, editor resolution from `$VISUAL`/`$EDITOR` (`code --wait`)
- **Safety**: removed eval-based prebuilt-binary downloader (x86_64-only),
  eval-free `freespace`, `cc` alias renamed `cch` (shadowed the C compiler),
  fixed the never-firing update check, non-blocking login (background fetch)
- **Project root**: `~/Development/code` via `$CODE_DIR` (gg/gb/freespace)
- **CI**: test suite (60 checks) runs on macos-latest + ubuntu-latest on
  every push; releases are published automatically from tags

## Removed

- WSL/Ubuntu/apt support, platform-detection layer, Ubuntu Docker test rig
- fnm (→ mise), direnv (→ mise `[env]`), `g` version manager (→ mise)
- `_cargo_smart_update` and prebuilt-binary download helpers
